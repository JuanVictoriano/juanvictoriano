Prioritized Recommendations

🔴 High Priority
Revoke FLS then delete 15 unused custom fields. All have 19-24 active grants — must revoke first. Start with 3 IT hardware fields and 2 duplicate fields (Department__c, Do_Not_Call__c).
Add an External ID field. Fan Master ID (10K records) or SFDC Fan ID (712 records) should be promoted to External ID type for fan data integrations.
Hide Business Fax and Data.com Key. Both at 0% with 76 FLS grants — the most visible empty fields on Contact. Revoke FLS immediately.
🟡 Medium Priority
Restrict FLS on 73 over-permissioned fields. Contact Role (81 grants, 1%) and Software License (81 grants, 1%) are the worst offenders. Limit to relevant profiles.
Convert Title and Department (Standard) to Picklist. Both have 76 grants (high visibility) with free text creating inconsistent data across 14K and 3K records respectively.
Enable State/Country Picklists for Mailing State, Mailing Country. Both are standard text fields with 76 grants — users enter free-form country/state names.
🟢 Low Priority
Consolidate 37 permission sets into groups. 5 Contact-specific CRUD sets plus cross-object sets create maintenance overhead. Permission Set Groups would simplify.
Audit subscriber ID fields. Shopify, Google, Apple, Feverup IDs all have 19 grants (limited visibility). Confirm which integrations are active and whether unused IDs should be hidden.
Remove IT hardware fields (Laptop, Phone Serial Number, Hardware) from Contact — same fields exist on Account at 0%. These belong in an IT asset management tool.
