Intermediate Module Org Development Model
Learn the org-based development workflow with Git — creating branches per developer, retrieving changes from sandboxes, committing to GitHub, and resolving merge conflicts before deploying.

💡 Recommended Order
Follow the phases sequentially — Phase 1 gets your local toolchain working (VS Code, SF CLI, Git), and Phase 2 builds on that with SFDX project structure and source-tracked deployments. Don't skip Phase 1 even if you've used VS Code before — the Salesforce Extension Pack and CLI authorization setup are their own thing.

Already comfortable with VS Code + CLI? Skip Phase 1 and jump directly to Phase 2's Quick Start: Salesforce DX to get hands-on with source-tracked orgs right away.

All links go directly to trailhead.salesforce.com. You'll need a free Trailhead account to track progress and earn points. Use a fresh Trailhead Playground for each phase's hands-on challenges.
